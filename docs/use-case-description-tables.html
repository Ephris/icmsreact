Table 1: Login Use Case Description

FieldDescription
NameLogin
IDUC01
ActorsAdmin, Student, Faculty Advisor, Company Supervisor, Coordinator, Company Admin, Department Head
DescriptionAllow a person with an account to sign in so they can access their personal dashboard and actions.
PriorityHigh
Pre-conditionThe sign-in page loads and the person has valid credentials.
Post conditionThe person is signed in and shown the correct home/dashboard for their role.
Basic Course of Action
StepActor ActionSystem Response
1Open the login page—
2—Show the login form with email and password fields
3Enter email and password—
4Click the login button—
5—Check credentials against database
6—If valid, create session and navigate to appropriate dashboard based on role
7Use Case ends—
Alternate course of action5.1 If credentials are wrong, show error message and let the person try again.
6.1 If account is inactive/deactivated, show message "Your account has been deactivated."
IncludeNone
Note/IssuesDisplay clear error messages. Role-based redirect to appropriate dashboard.

Table 2: Logout Use Case Description

FieldDescription
NameLogout
IDUC02
ActorsAll signed-in users
DescriptionTo disable the permission of accessing the account and end the current session.
PriorityHigh
Pre-conditionThe user must be successfully logged in to the system.
Post conditionThe permission of accessing that account is blocked until the user logs in again.
Basic Course of Action
StepActor ActionSystem Response
1Click on logout button from account menu—
2—Invalidate the current session
3—Regenerate CSRF token
4—Redirect to home page
5Use Case ends—
Alternate course of action1.1 If user has unsaved work, system may warn about unsaved changes.
IncludeLogin
Note/IssuesSession is completely cleared on logout.
